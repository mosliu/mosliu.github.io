在Hostinger空间上搭建gitserver

Posted on 2017-02-09

缘起

两年前买了一个Hostinger空间。ping值在110左右，还行吧。。

前一段偶尔发现空间可以支持git命令

遂想到，可以做自己的git服务器。大善大善。

虽然只能用自己的用户名，但是还是自己的服务器啊，挺好，挺好。

步骤

  1. 服务器端，建立一个文件夹，假设叫git/my.git

  2. 创建仓库

    系统提示：

    Initialized empty Git repository in /home/u55555555555/git/my.git/

    其中的u55555555555 是你的用户名

    这样就建好了。

  3. 使用
    回到windows，直接迁出就可以了

    其中ip地址为你的ssh登录的地址，60000是你的端口号

  4. 基本应用，献丑了。

补充

  • 经体验，是可以使用key登录的，不用每次输密码了！

    1. 在home（~）下面 新建文件夹
    2. 进入.ssh后，新建文件touch authorized_keys

    3. 设置文件夹和文件的权限

    4. 导入你的公钥，假设放在了/tmp下的id_rsa.pub

新建一个托管在Github上的Hexo博客（windows平台）

Posted on 2017-02-08

安装时碰到几个要点

从头到尾写一下

1. 安装git 、nodejs

不细写了。自己google吧。很简单的。

  1. （其实是很早就安好了，没再安，嘿嘿）

2. 安装hexo

参考 hexo.io网站。

3. Github配置

  • 建立github.io仓库

    假设你的用户名是mosliu
    那么新建一个名字为mosliu.github.io的repository
    就可以直接通过mosliu.github.io访问了

  • 生成public key对

    通过命令ssh-keygen -t rsa -C "XXX@gmail.com"生成私钥和公钥，XXX@gmail.com替换成自己的邮箱
    这里我有个惨痛的教训，我电脑上装了好几套linux指令集。然后生成时候，直接在cmd下生成了，结果后期部署的时候就会出问题！后来找了半天发现 原因是调用的git和我默认的git不是一个啊不是一个。其实主要问题就是秘钥存放位置问题。这里建议使用git bash生成。

    生成后，将公钥添加到github里面。

  • 测试

    使用ssh -T git@github.com

    如果看到

    就成功了

  • 设置username和email

4. 查看本地

访问127.0.0.1:4000能看到，里面怎么新建 网页都有的。

5. 发布
